# Техническое задание
## "Базовый планировщик процессов на Perl"

### 1. Общие положения
1.1. Цель разработки: Создание простого планировщика задач с поддержкой многозадачности
1.2. Назначение: Управление выполнением задач по расписанию

### 2. Функциональные требования
2.1. Основные функции:
- Создание задач с указанием времени выполнения
- Отмена запланированных задач
- Просмотр списка задач
- Обработка ошибок
- Логирование

2.2. Алгоритмы планирования:
- FCFS
- Round Robin
- Приоритетное планирование

### 3. Технические требования
3.1. Требования к ПО:
- Perl 5.30+
- Модуль DateTime
- Модуль POSIX
- Модуль Log::Log4perl

3.2. Системные требования:
- Любая POSIX-совместимая ОС
- 256 МБ ОЗУ
- 50 МБ свободного места на диске

### 4. Интерфейс
4.1. Консольный интерфейс:
- Добавление задачи (add)
- Просмотр списка (list)
- Отмена задачи (cancel)
- Изменение приоритета (priority)
- Выход (exit)

4.2. Формат вывода:
- Табличное представление списка задач
- Детальная информация о задачах
- Логирование в файл

### 5. Требования к надежности
5.1. Обработка ошибок:
- Некорректный ввод
- Проблемы с доступом к файлам
- Ошибки выполнения задач

5.2. Восстановление:
- Сохранение состояния
- Автовозобновление после сбоев

### 6. Документация
6.1. Должна содержать:
- Руководство пользователя

- Примеры использования
- Инструкции по установке

### 7. Этапы разработки

### Глоссарий
- FCFS (First-Come, First-Served) - алгоритм планирования, при котором задачи выполняются в порядке поступления
- Round Robin - циклический алгоритм планирования с квантованием времени
- Приоритетное планирование - алгоритм, учитывающий важность задач

### Примечания
- Использовать объектно-ориентированный подход
- Применять модульный принцип построения
- Обеспечить кроссплатформенность
- Предусмотреть возможность расширения функционала
